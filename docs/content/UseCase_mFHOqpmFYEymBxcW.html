Scenario
Steps
1. El Arrendador1 autenticado accede al sistema.
2. El Arrendador1 selecciona la opción "Registrar propiedad".
3. SYSTEM muestra un formulario para el registro de la propiedad.
4. El Arrendador1 ingresa la información solicitada y carga las imágenes de la propiedad.
5. El Arrendador1 revisa los datos y confirma el registro.
6. SYSTEM valida la información suministrada.
7. if Si las validaciones son correctas.
    7.1. SYSTEM guarda la información y registra la propiedad.
8. else
    8.1. SYSTEM no permitirá guardar la información y mostrara los campos a incorrectos o incompletos
   end if
9. SYSTEM muestra un mensaje de confirmación: "La propiedad ha sido registrada exitosamente".
10. La propiedad queda visible públicamente para los arrendatarios en las búsquedas y listados.

Extension
Steps
3.a. formulario tendra campos obligatorios y opcionales.
    1. Obligatorios: al menos tres imágenes, precio, dirección, tipo de propiedad.
    2. Opcionales: condiciones (ejemplo: depósito, plazo).
6.a. Al validar la información se debe validar.
    1. Que las imágenes hayan sido cargadas.
    2. Que el precio, la dirección y el tipo de propiedad no estén vacíos.
    3. Que los formatos de datos sean correctos (ejemplo: precio numérico, dirección válida).
